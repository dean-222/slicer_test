# 03. 워크플로우

## `FemurFracturePlannerWorkflow.py`의 역할

이 파일은 사용자 작업 흐름을 담당합니다.

쉽게 말하면:

- 버튼을 누르면 무엇을 할지
- 노드를 바꾸면 무엇을 갱신할지
- 오류가 나면 어떤 메시지를 띄울지

를 모아 둔 파일입니다.

## 주요 흐름별 메서드

### 1. 입력 볼륨 관련

#### `onInputVolumeChanged()`

- 입력 CT 볼륨이 바뀌면
  - Segment Editor source volume 갱신
  - segmentation 연결 보정
  - slice viewer 갱신
  - volume rendering 갱신
  - rotation slider 초기화

#### `onLoadVolumeClicked()`

- 파일로부터 CT 볼륨을 불러온다.

#### `onLoadDicomClicked()`

- Slicer DICOM browser를 연다.

### 2. 볼륨 표시 및 렌더링

#### `onVolumeVisibilityToggled()`

- 2D slice viewer 배경에 볼륨을 보이거나 숨긴다.

#### `onVolumeRenderingToggled()`

- 3D volume rendering 표시를 켜고 끈다.

#### `onThresholdSliderChanged()`

- 볼륨 렌더링 threshold를 실시간 반영한다.

### 3. 회전 제어

#### `getSelectedRotationAxis()`

- 현재 UI에서 선택된 회전 축을 반환한다.

#### `onRotationAngleChanged()`

- 슬라이더 값에 따라 볼륨/세그멘테이션/골편의 공통 transform을 회전시킨다.

#### `onResetRotationClicked()`

- 회전을 0도로 초기화한다.

#### `onRotate90Clicked()`

- 현재 각도에 90도를 더한다.

### 4. 세그멘테이션 및 골편 분리

#### `onSegmentationNodeChanged()`

- 상단 세그멘테이션 선택기와 Segment Editor를 동기화한다.

#### `onEditorSegmentationNodeChanged()`

- Segment Editor 내부 변경 사항을 상단 UI에 다시 반영한다.

#### `onSeparateFragmentsClicked()`

- 현재 segmentation에서 연결 성분을 분리해 골편 모델을 만든다.
- 내부적으로 `logic.separateBoneFragments()`를 호출한다.

### 5. 가이드 모델 관련

#### `onLoadGuideClicked()`

- 외부 guide bone model 파일을 불러온다.

#### `onMirrorGuideClicked()`

- guide model을 X축 기준으로 미러링한다.

#### `onGuideVisibilityToggled()`

- guide model 표시를 켜고 끈다.

#### `onGuideModelChanged()`

- 현재 guide model 상태를 버튼 표시와 동기화한다.

### 6. 자동 정렬

#### `onRunIcpReductionClicked()`

- 분리된 골편들을 guide model에 ICP 정합한다.

#### `onRunSurfaceSnapClicked()`

- 골편 두 개의 가까운 골절면을 기준으로 snap 정렬을 수행한다.

### 7. Scene 반응

#### `onNodeAdded()`

- 새 volume이 들어오면 자동 선택
- 새 fragment model이 생기면 표 갱신

#### `onNodeRemoved()`

- fragment model 삭제 시 표 갱신

## 이 파일을 읽는 포인트

- “버튼을 눌렀는데 어디서 처리되지?”를 찾을 때 가장 먼저 본다.
- 계산식 자체보다 사용자 흐름을 이해할 때 유용하다.
