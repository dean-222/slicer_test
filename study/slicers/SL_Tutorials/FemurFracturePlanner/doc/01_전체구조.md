# 01. 전체 구조

## 모듈의 큰 흐름

이 모듈은 대퇴골 골절 계획을 위한 3D Slicer Scripted Module입니다.

사용자 흐름은 대체로 아래 순서로 진행됩니다.

1. CT 볼륨을 불러온다.
2. 볼륨을 2D/3D에서 확인한다.
3. Segment Editor로 뼈 영역을 분할한다.
4. 분할 결과를 기반으로 골편 모델을 생성한다.
5. 가이드 모델을 불러오거나 미러링한다.
6. ICP 또는 surface snap으로 골편을 정렬한다.
7. Fragment Manager에서 골편 상태를 관리한다.

## 파일 간 관계

### 1. 진입점

`FemurFracturePlanner.py`

- Slicer가 모듈을 로드할 때 가장 먼저 참조하는 파일이다.
- 모듈 메타데이터를 등록한다.
- 테스트 클래스도 여기 있다.

### 2. UI 중심 컨트롤러

`FemurFracturePlannerWidget.py`

- 실제 팝업 창을 만든다.
- `.ui` 파일을 로드한다.
- UI 이벤트를 각 메서드에 연결한다.
- `WorkflowMixin`, `FragmentManagerMixin`, `Logic`를 묶는 중심 허브 역할이다.

### 3. 사용자 작업 흐름

`FemurFracturePlannerWorkflow.py`

- 사용자가 버튼을 누르거나 노드를 바꿀 때 반응하는 메서드가 모여 있다.
- “무엇을 눌렀을 때 어떤 순서로 동작할지”를 담당한다.

### 4. 계산 로직

`FemurFracturePlannerLogic.py`

- VTK/Slicer API를 사용한 실제 처리 로직이 들어 있다.
- 렌더링 조정, 골편 분리, 미러링, ICP 정합, fracture surface snap 같은 계산이 여기에 있다.

### 5. 골편 관리 UI

`FemurFracturePlannerFragmentManager.py`

- 생성된 골편 목록을 표로 보여 준다.
- 표시 여부, 이동 gizmo, 색상 변경, 삭제, 좌표/회전 추적을 담당한다.

## 현재 구조의 장점

- `Widget` 파일이 너무 커지는 문제를 줄였다.
- UI 제어와 계산 로직이 분리되어 읽기 쉬워졌다.
- 이후 기능별 추가 분리가 가능하다.
