# 04. 로직 계층

## `FemurFracturePlannerLogic.py`의 역할

이 파일은 실제 계산과 데이터 처리를 담당합니다.

UI와 직접 연결된 코드가 아니라, “무엇을 계산할지”에 집중한 계층입니다.

## 핵심 클래스

### `FemurFracturePlannerParameterNode`

- UI 상태를 MRML parameter node로 다루기 위한 래퍼
- 현재는 `inputVolume` 중심으로 사용한다.

### `FemurFracturePlannerLogic`

- 실제 처리 함수들이 모여 있는 핵심 클래스

## 주요 기능

### 1. Volume Rendering

#### `showVolumeRendering(volumeNode, threshold)`

- volume rendering display node를 준비하고 표시한다.
- preset으로 `CT-Bone`을 사용한다.

#### `hideVolumeRendering(volumeNode)`

- volume rendering display node를 제거한다.

#### `adjustVolumeRenderingThreshold(volumeNode, threshold)`

- opacity function을 다시 구성해 threshold에 따른 bone 가시성을 조정한다.

### 2. 회전 처리

#### `rotateVolume(volumeNode, axis, angle)`

- 입력 볼륨에 연결된 transform을 회전시킨다.
- 동시에 segmentation node와 fragment model에도 같은 transform을 연결한다.

즉, 단순히 볼륨만 도는 게 아니라 관련 객체가 같이 회전하도록 맞춰 둔 구조다.

### 3. 골편 분리

#### `separateBoneFragments(segmentationNode, referenceVolumeNode=None)`

- segmentation의 visible segment들을 합친 뒤
- connectivity filter로 연결 성분을 분리하고
- 큰 영역 위주로 fragment model을 생성한다.

현재 구현 포인트:

- visible segment만 사용
- 빈 geometry는 예외 처리
- 가장 큰 상위 영역 위주로 골편 생성
- 생성된 골편에 transform 상속 가능

### 4. 모델 미러링

#### `mirrorModel(sourceModelNode)`

- X축 기준으로 scale `(-1, 1, 1)` 미러링
- 법선 방향을 다시 보정
- 새로운 `_Mirrored` 모델 노드 생성

### 5. ICP 자동 정렬

#### `runIcpRegistration(fragmentModelNodes, guideModelNode, logCallback=None)`

- **병합 ICP 정합 (Combined ICP)**: 각 골편을 따로 정합할 때 골절 단면이 어긋나는 한계를 해결하기 위해, 모든 골편의 월드 좌표 메쉬를 수집하여 하나의 단일 메쉬로 병합(`vtkAppendPolyData`)한 후 가이드에 통째로 ICP를 적용합니다.
- 계산된 단일 공통 변환 행렬을 모든 골편의 개별 변환 노드에 나누어 대입하여, 골편 간 상대적 맞물림 관계를 100% 보존하며 가이드에 맞게 전체 축 정렬을 완료합니다.

이 점이 중요하다:

- 골절선이 어긋나지 않고 유지된다.
- 원본 메쉬를 직접 수정하지 않고 4x4 부모 변환 노드 행렬을 갱신한다.
- 상위 transform 계층이 있어도 역행렬 처리를 통해 계산을 이어갈 수 있다.

### 6. Fracture Surface Snap

#### `runFractureSurfaceSnap(frag1Node, frag2Node, logCallback=None)`

- 두 골편의 마주보는 잘린 단면(골절면)을 분석하여, 첫 번째 골편을 두 번째 골편에 단면 대 단면으로 정교하게 정복 결합시킨다.
- 원샷 최소제곱 강체 정합인 Landmark 변환(`vtkLandmarkTransform`)을 통해 골편을 정합하고, 그 결과 행렬을 변환 노드(`vtkMRMLLinearTransformNode`)에 적용한다. (포인트 클라우드만 있는 랜드마크 데이터에 2-Stage ICP 구동 시 연산 붕괴가 발생하므로, 안정적인 단일 LandmarkTransform 구조로 고정함)

핵심 설계 및 오정합 방지 알고리즘:

- **고립 정점 제거 전처리 (`vtkCleanPolyData`)**: 골편 분리 메쉬 내부에 겉보기엔 안 보이지만 꼭짓점 데이터 상에 남아있던 고립된 쓰레기 정점들을 완벽히 정리하여, 바운딩 박스(Bounds)와 중심 좌표(Center)가 실제 뼈 조각 크기로 올바르게 정밀 축소되도록 보정합니다.
- **끝단 마진(20mm) 공간 필터링**: 두 골편의 Z축 높이 범위를 측정하여, 실질적으로 잘린 단면이 존재하는 마주보는 끝단 경계(20.0mm 이내) 부근 영역만 정합 대상으로 필터링하여 옆구리(Cortex) 표면이 오정합되는 현상을 원천 차단합니다.
- **법선 벡터 내적 필터링**: 두 골편 단면의 표면 법선 벡터가 서로 정반대로 마주보고 있는 영역(**내적 -0.6 이하**로 한층 엄격하게 강화)만 유효한 골절면으로 인식합니다.
- **장축 정렬 방향성 필터링**: 뼈의 장축(Z축)에 더 똑바른 단면 정점들만 선별하기 위해 법선 벡터의 Z축 성분 크기 기준을 **0.4 이상**으로 강화하여 아웃라이어 유입을 완전 차단합니다.
- **단면 무게중심 기준 초기 오프셋 보정 (Pre-alignment)**: 두 단면 영역의 무게중심을 먼저 일치시킨 가상 공간에서 KD-Tree 최근접 탐색을 수행합니다. 이를 통해 골편들이 수평(X-Y) 또는 수직(Z) 방향으로 많이 어긋나 있는 초기 상태에서도 엉뚱한 옆구리에 쏠리지 않고 정확하게 마주보는 대칭 단면 점들을 1:1로 매칭할 수 있습니다.
- **진행 피드백 로깅**: 실시간 정합 최종 오차(Mean Distance, mm) 및 변환 델타 행렬(Translation vector)을 진행 로그창에 상세히 리포팅합니다.

## 이 파일을 수정해야 하는 경우

- 정렬 알고리즘을 바꿀 때
- connectivity 기준을 바꿀 때
- threshold/opacity 규칙을 바꿀 때
- geometry/transform 계산을 고칠 때

## 읽을 때 주의할 점

- 여기에는 UI 이벤트가 거의 없다.
- 대신 VTK transform, matrix, polydata 처리 흐름을 따라가야 한다.
