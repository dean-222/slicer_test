# 05. 골편 관리

## `FemurFracturePlannerFragmentManager.py`의 역할

이 파일은 생성된 골편 모델을 표 형태로 관리하는 UI 계층입니다.

주요 대상은 이름이 `Femur_Fragment*` 로 시작하는 모델 노드입니다.

## 주요 기능

### 1. 골편 목록 표 구성

#### `updateFragmentTable()`

- scene에서 `Femur_Fragment*` 모델을 찾는다.
- 표의 각 행에 아래 정보를 구성한다.
  - 이름
  - 표시 여부
  - 이동 gizmo 활성화 여부
  - 색상 버튼
  - 삭제 버튼

### 2. 이름 변경

#### `onFragmentTableItemChanged()`

- 표에서 이름을 바꾸면 실제 MRML node 이름도 바꾼다.

### 3. 표시 토글

#### `onFragmentVisibilityToggled()`

- 골편 display node visibility를 켜고 끈다.

### 4. 색상 변경

#### `onFragmentColorClicked()`

- 색상 대화상자를 열고
- 선택한 색을 display node에 반영한다.

### 5. 골편 삭제

#### `onFragmentDeleteClicked()`

- 선택한 골편 모델 노드를 scene에서 제거한다.

### 6. 이동 gizmo

#### `onFragmentInteractionToggled()`

- 골편별 linear transform node를 준비한다.
- transform display의 editor visibility를 켜서 gizmo를 노출한다.

중요한 포인트:

- 기존에 부모 transform이 있으면 계층을 가능한 유지하려고 한다.
- 즉, 무조건 새 transform으로 덮어쓰는 게 아니라 기존 관계를 고려한다.

### 7. 일괄 삭제

#### `onClearModelsClicked()`

- `Femur_Fragment*`
- `_Mirrored`

에 해당하는 모델 노드를 한 번에 정리한다.

### 8. 선택 골편 추적

#### `onFragmentTableSelectionChanged()`

- 현재 선택된 골편을 기준으로
  - 이름
  - 위치
  - 회전

표시 영역을 갱신한다.

#### `onActiveTransformNodeModified()`

- transform 수정 이벤트를 받아 실시간으로 위치/회전을 업데이트한다.

#### `removeActiveTransformObserver()`

- 이전 선택 골편 observer를 정리한다.

## 이 파일이 중요한 이유

정렬 알고리즘은 `Logic`에 있지만, 사용자가 실제로 만지는 마지막 인터페이스는 여기인 경우가 많습니다.

즉:

- 골편이 생성됐는지 확인
- 개별 골편을 켜고 끄기
- 수동으로 만지기
- 좌표 확인하기

같은 작업은 대부분 이 파일을 통해 이뤄집니다.
