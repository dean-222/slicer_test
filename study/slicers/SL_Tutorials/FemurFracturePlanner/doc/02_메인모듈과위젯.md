# 02. 메인 모듈과 위젯

## `FemurFracturePlanner.py`

이 파일은 “모듈 등록 파일”에 가깝습니다.

### 핵심 클래스

#### `FemurFracturePlanner`

- Slicer 모듈 목록에 표시될 제목, 카테고리, 설명을 등록한다.
- 실제 기능 구현은 여기보다 다른 파일에 있다.

#### `FemurFracturePlannerTest`

- 최소 기능 검증용 테스트 클래스다.
- 주로 아래 두 가지를 확인한다.
  - guide model 미러링이 되는지
  - ICP 정합이 동작하는지

### 이 파일을 수정해야 하는 경우

- 모듈 이름, 설명, 카테고리를 바꿀 때
- 기본 테스트를 보강할 때
- 모듈 로딩/등록 방식이 바뀔 때

## `FemurFracturePlannerWidget.py`

이 파일은 “메인 컨트롤 타워”에 가깝습니다.

### 핵심 클래스

#### `PlannerDialog`

- 팝업 전용 `QDialog`
- 닫기 버튼을 눌렀을 때 완전히 종료하지 않고 숨기는 역할을 한다.

#### `FemurFracturePlannerWidget`

- 전체 UI의 메인 위젯 클래스
- 아래 역할을 동시에 수행한다.
  - 팝업 창 생성 및 가상 정복 패널 내에 QTextEdit 기반의 **실시간 진행 로그창(Progress Log)**을 동적으로 생성 및 삽입
  - `.ui` 파일 로드
  - Segment Editor 내장
  - MRML scene observer 등록
  - parameter node 연결
  - workflow/fragment manager 메서드 연결

### 중요한 멤버 변수

- `self.logic`
  - 계산 로직 객체
- `self.plannerDialog`
  - 독립 팝업 창
- `self.segmentEditorWidget`
  - 내장 Segment Editor 위젯
- `self.segmentEditorNode`
  - Segment Editor 상태를 저장하는 MRML 노드
- `self._parameterNode`
  - 모듈 UI 상태를 연결하는 parameter node
- `self._activeTransformNode`
  - 선택 골편 추적용 transform

### 중요한 메서드

#### `setup()`

- 팝업 창 생성
- `.ui` 로드
- 버튼/슬라이더/선택기 signal 연결
- Segment Editor 준비

#### `enter()`

- 모듈 진입 시 필요한 상태 재연결
- Scene 리로드 후에도 다시 동작하도록 보정

#### `exit()`

- observer 해제
- editor 연결 정리

#### `cleanup()`

- reload나 종료 시 리소스를 정리한다.

### 이 파일을 수정해야 하는 경우

- UI 생성 방식 자체를 바꿀 때
- signal 연결 구조를 바꿀 때
- 팝업 동작 방식, editor 생명주기를 바꿀 때
