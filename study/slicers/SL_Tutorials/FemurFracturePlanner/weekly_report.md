# 대퇴골 골절 계획 플래너 - 주간 작업 보고서 (월~목)

본 문서는 2026년 6월 22일(월)부터 25일(목)까지 진행된 FemurFracturePlanner 모듈의 주요 기능 개선 및 버그 수정 사항을 요일별로 요약한 주간 작업 보고서입니다.

---

## 📅 화요일 (6월 23일) : 아키텍처 대공사 및 뼈대 구축
이 날은 하나의 거대한 파일에 뭉쳐있던 코드를 역할별로 쪼개고, 기본적인 UI와 로직을 복원하는 기틀을 다진 날입니다.

- **모듈 구조 분할(MVC화)**: `FemurFracturePlanner` 메인 파일에서 복잡한 화면 단은 `Widget.py`로, 무거운 계산 로직은 `Logic.py`로 각각 독립시켜 향후 유지보수가 쉽도록 대공사를 진행했습니다.
- **UI 및 기본 기능 복원**: 영상 처리 영역의 UI와 버튼들을 새로 추가하고, 한동안 막혀있던 영상 반전 및 3D 엣지 검출 등의 기초 로직들을 다시 연결해 복원했습니다.
- **창 크기 최적화**: 팝업이 이상하게 뜨던 문제를 해결해 고정 높이 확장 모드로 깔끔하게 나오도록 다듬었습니다.

## 📅 수요일 (6월 24일) : 안정성 강화 및 3D 영상처리 고도화
화요일에 다진 뼈대를 바탕으로, 치명적인 오류들을 잡아내고 3D 영상 처리 알고리즘의 퀄리티를 대폭 끌어올린 날입니다.

- **리로드(Reload) 아키텍처 완성**: 코드를 수정하고 Slicer에서 Reload 버튼을 눌러도 예전 코드가 실행되던 끔찍한 캐시 꼬임 문제를 완전히 해결했습니다. 이제 하위 파일들까지 싹 갱신되는 순차 리로드 로직이 적용되었습니다.
- **마스킹 엣지 검출 (노이즈 제거)**: 뼈만 깔끔하게 분리하기 위해, 임계값 미만 영역을 공기(-1000 HU)로 완전히 덮어버린 뒤 엣지를 추출하는 정교한 마스킹 로직을 도입했습니다.
- **언더플로우(데이터 깨짐) 버그 해결**: CT 볼륨 값을 변환할 때 음수 값이 깨지는 현상을 막기 위해 안전한 타입 캐스팅(`vtkImageCast`) 계층을 추가했습니다.
- **하드코딩 분리**: 코드 중간중간 숨어있던 수치들(반복 횟수, 임계값 등)을 최상단 상수로 일괄 분리해 언제든 쉽게 조율할 수 있게 만들었습니다.

## 📅 목요일 (6월 25일) : 핵심 알고리즘 교체 및 디테일·버그 박멸
고급 정합 알고리즘을 도입하고, 사용하면서 불편했던 자잘한 UI 버그와 시각적 요소들을 완벽히 다듬은 날입니다.

- **골절면 스냅(Snap) 알고리즘 고도화**: 단 1회만 대충 위치를 맞추던 낡은 연산을 뜯어내고, 수십 번 반복하여 뼈 단면을 톱니바퀴처럼 정밀하게 맞물리게 하는 **ICP(반복적 최근접점) 수학 알고리즘**을 심었습니다.
- **스냅 조기종료(과적합 방지)**: 뼈가 너무 미끄러져 파고들지 않도록 최대 반복 50회 제한 및 0.5mm 오차 달성 시 즉시 연산을 멈추는 브레이크를 달았습니다.
- **시각적 편의성 (색상 개선)**: 분리된 골편들이 칙칙한 색으로 나오는 대신, 연녹색, 주황색, 파란색 등 **눈에 확 띄는 고대비 색상**으로 고정 배정되게 만들었습니다.
- **가이드 골편 메모리 누수 해결**: '전체 삭제'를 눌러도 원본 가이드 뼈 모델이 지워지지 않고 투명하게 씬에 쌓이던 누수 버그를 역추적하여 완벽히 지워지도록 고쳤습니다.
- **UI 동기화 버그 일괄 수정**: 
  - 골편 목록 표의 색상과 실제 3D 뼈 색상이 다르게 나오던 현상 해결
  - 가이드 숨기기 버튼이 안 눌리던 현상 해결
  - 골편 분리 성공 후 표 갱신이 무시되던 현상 해결
  - 모듈 전반의 영어 독스트링 및 주석을 알기 쉬운 한글로 전면 번역
